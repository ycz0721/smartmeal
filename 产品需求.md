# SmartMeal 产品需求文档（PRD）

> 版本：v0.6.8 | 更新日期：2026-05-13

---

## 一、产品概述

### 1.1 产品定位

SmartMeal 是一款 **AI 驱动的家庭膳食计划工具**。用户告诉 AI 想吃什么，系统自动生成一周菜单，并智能计算购物清单（减去冰箱已有食材），解决"今晚吃什么"的决策疲劳。

### 1.2 核心价值

- **省时间**：AI 代劳菜单规划，不用每天纠结
- **省金钱**：购物清单精准计算，只买真正缺的，减少食物浪费
- **个性化**：记住你的口味偏好、饮食限制、不耐受食材
- **零门槛**：手机号注册即用，移动端优先

### 1.3 目标用户

- 每天纠结"吃什么"的个人用户
- 需要规划多人多天饭菜的家庭主妇/主夫
- 想尝试新菜但不知从何下手的烹饪新手
- 有特定饮食限制（过敏、素食等）需要避开某些食材的人

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 全栈框架 | Next.js 15 (App Router) | 前后端一体，API 路由 + 页面渲染 |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS + shadcn/ui | 原子化 CSS + UI 组件库 |

| 部署 | 阿里云 ECS (Ubuntu 22.04) | PM2 进程管理，Nginx 反向代理 |
| 版本管理 | Git + GitHub | 代码托管，Git-flow 简化版 |

## 四、非功能需求

### 3.1 性能

- AI 接口超时：30 秒
- 页面首次加载：3 秒内可交互（移动端 4G 网络）
- 生成一周计划：30 秒内完成（含 AI 调用）

### 3.2 安全

- 密码 bcrypt 加密存储
- API 路由全部需要登录验证
- 手机号脱敏显示（138****5678）

### 3.3 可用性

- 移动端优先设计，响应式布局
- 关键操作有 loading 状态和错误提示
- 空状态有引导文案
- 骨架屏加载占位

### 3.4 技术约束

- 部署于阿里云 ECS（Ubuntu 22.04，IP: 8.134.146.192）
- 数据库：PostgreSQL + Prisma ORM
- AI 依赖：DeepSeek API（文本生成）+ 智谱 GLM-4V（图像识别）
- 图片服务：上传图片通过 API 路由提供（非静态文件）
